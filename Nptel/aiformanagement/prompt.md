# **1. CONCEPTS (simple, complete, short)**

## **Week 1**

### **AI Types**

* **ANI (Artificial Narrow Intelligence):** Does one specific task (Siri, Netflix).
* **AGI:** Human-level intelligence (not achieved yet).
* **ASI:** Beyond human intelligence (theoretical).

### **AI Winter**

* Period where AI funding + interest dropped due to overpromises and failures.

### **AI in Strategy**

* AI creates advantage by:

  * Personalization at scale
  * Learning from data (feedback loops)
  * Automating complex tasks

### **Learning Types**

* **Supervised:** Has labels → predicts output
* **Unsupervised:** No labels → finds patterns
* **Reinforcement:** Learns via reward

### **AI Hallucination**

* AI gives confident but wrong answers.

### **Reasoning**

* **Deductive:** General → specific (guaranteed correct if premise true)
* **Inductive:** Specific → general (not always certain)

### **Neuro-symbolic AI**

* Combines neural networks + logical reasoning.

### **AI Bias**

* AI can be biased due to biased data.

### **AI Deployment Insight**

* Success needs:

  * Training users
  * Process optimization
  * Not just building model

---

## **Week 2**

### **ML Limitations**

* Easily fooled by small input changes
* Needs supervision
* Lacks common sense

### **Overfitting**

* Model memorizes noise → poor generalization

### **Supervised Learning Characteristics**

* Uses labeled data
* Used in classification/regression
* Learns input → output mapping

### **Perceptron Limitation**

* Works only if data is linearly separable

### **Backpropagation**

* Solves multilayer learning
* Updates weights using error

### **Error Function**

* Half squared error between predicted and actual

### **Hidden Layers**

* More layers → more power
* But increases overfitting risk

### **ANN Advantage**

* Hidden layers → capture complex patterns

### **Time Series Components**

* Trend
* Seasonality
* Cyclicity
* Randomness

### **White Box vs Neural Network**

* White box → interpretable
* Neural network → black box

# **1. Concepts (simple, full coverage)**

### **1. Business Value of AI**

AI is useful only when it creates measurable value like cost savings or revenue increase. Just building a model is not enough—business impact matters.

### **2. AI Value Drivers**

AI creates value in 4 ways:

* **Automation** → replaces human work (reduces cost)
* **Anticipation** → predicts future (better decisions)
* **Coordination** → improves processes between teams
* **Personalization** → custom experience for users

### **3. Resource-Based View (RBV)**

Just owning AI tech is useless. You need to convert it into **capabilities** using people, processes, and data. Valuable resources must be **valuable, rare, and hard to copy**.

### **4. Business Model (Supply Side)**

Business model explains how a company:

* creates value
* delivers value
* earns money

It focuses on the **supply side**, not customer demand.

### **5. Transaction Cost Economics (TCE)**

Firms choose between:

* doing work internally
* outsourcing

Decision depends on **total cost = production cost + transaction cost**.

### **6. Transaction Costs**

These include:

* searching vendors
* negotiating contracts
* monitoring performance

Also includes **risk (bias, failure, etc.)**

### **7. AI-as-a-Product**

Using external AI adds:

* coordination cost
* contract enforcement cost

### **8. Economies of Scale**

When production increases:

* cost per unit decreases

### **9. AI Capability Components**

AI capability =

* Technology
* People
* Data
* Culture
* Training & incentives

### **10. Performance Measurement**

To check AI success:

* Operational metrics → speed, efficiency
* Business metrics → revenue, profit
* Technical metrics → accuracy

### **11. Key KPIs in AI Systems**

Examples:

* Escalation rate
* Resolution time
* Sentiment score

# **1. Concepts (simple + complete, 2–3 lines each)**

### **1. Financial vs Technical vs Operational Metrics**

* Technical metrics (Precision, Recall, AUC) tell how well the model works mathematically.
* Operational metrics measure process efficiency (speed, automation).
* Financial metrics (revenue, cost savings, CLV) show real business value → most important for executives.

---

### **2. A/B Testing (Randomized Controlled Trial)**

* Users are randomly split into two groups: control vs treatment.
* Only one change is applied → ensures fair comparison.
* Considered **gold standard** because it removes bias and proves causality.

---

### **3. Synthetic Control Method (SCM)**

* Used when A/B testing is not possible.
* Creates a **fake control group** using weighted combination of other groups.
* Helps estimate “what would have happened” without intervention.

---

### **4. Measuring AI Business Value**

* Biggest challenge: proving ROI to non-technical stakeholders.
* Need **clear, measurable impact** (money, efficiency, outcomes).
* Without this → AI projects fail to scale.

---

### **5. Class Imbalance Problem**

* Happens when one class is very rare (e.g., fraud detection).
* Accuracy becomes misleading (model predicts majority always).
* Use better metrics: Precision, Recall, AUC, KS.

---

### **6. AUC (Area Under Curve)**

* Measures model’s ability to distinguish classes.
* Higher AUC → better model across all thresholds.
* Useful when classes are imbalanced.

---

### **7. Precision & Recall**

* Precision: how many predicted positives are correct.
* Recall: how many actual positives are captured.
* Trade-off between both depending on problem.

---

### **8. KS Statistic**

* Measures separation between positive & negative distributions.
* Higher KS → better discrimination.
* Common in **credit scoring**.

---

### **9. Hierarchy of Metrics**

* Technical → Operational → Financial
* Business cares most about **financial outcomes**
* Managers focus on **operational metrics**

---

### **10. Operational Metrics Examples**

* Process efficiency
* Speed of delivery
* Automation of tasks
* These show practical system impact

---

### **11. Ranking Metrics (Search/Feed systems)**

* Used where order matters (top results important)
* Metrics:

  * Mean Reciprocal Rank (MRR)
  * NDCG (Normalized Discounted Cumulative Gain)

---

### **12. Mean Reciprocal Rank (MRR)**

* Focuses on position of first correct result
* Higher if correct answer appears early

---

### **13. NDCG**

* Rewards correct results higher in ranking
* Penalizes relevant results appearing lower
* Considers position importance

---

### **14. Swap Set Analysis (Credit Models)**

* Compares old vs new model decisions
* Uses:

  * LTV (good customers)
  * LGD (bad customers)
* Goal: maximize **net profit**

---

### **15. Financial Evaluation in AI**

* Best metric: **net profit under constraints**
* Not just accuracy → business impact matters

---

### **16. KS vs Accuracy**

* Accuracy depends on threshold
* KS evaluates across all thresholds
* Hence better for imbalanced datasets

* **Structured**: Routine, rules-based (easy to automate).
* **Semi-structured**: Mix of logic + human judgment.
* **Unstructured**: Complex, no fixed rules (requires thinking).

* Central storage of structured data.
* Used for analysis and reporting.
* Core part of BI systems.

* Helps understand how AI makes decisions.
* Important for **trust + transparency**.
* Tools: SHAP, LIME.

---

* Explain predictions by showing feature importance.
* Used when model is complex (black box).
* Helps stakeholders trust AI.

---

### **8. AI Assistant vs AI Agent**

* **Assistant**: Needs instructions, reactive.
* **Agent**: Autonomous, takes actions, plans workflows.

---

### **9. AI Agents**

* Can design workflows.
* Execute tasks independently.
* Use tools and memory.

---

### **10. Correlation vs Causation**

* Just because two things relate doesn’t mean one causes the other.

---

* Humans often biased.
* AI can also learn bias from data.
* Must validate before decisions.

---

### **12. Anchoring Bias**

* First information influences decisions heavily.
* Even if wrong, it affects thinking.

---

### **13. Confirmation Bias**

* People look for info that supports their beliefs.
* Ignore conflicting evidence.

---

### **14. Governance in AI**

* Focuses on fairness, transparency, accountability.
* Important in high-risk domains.

---

### **15. Model Simplicity vs Complexity**

* Simple models → easier to trust.

* Helps justify decisions.

# **1. Concepts (simple, full coverage)**

### **1. Feedforward Neural Networks (FNN)**

* Data moves only in one direction (input → output).
* No memory of past inputs.
* Used for basic classification/regression.

---

### **2. Vanishing Gradient Problem**

* In deep networks, gradients become very small during backpropagation.
* Earlier layers learn very slowly or stop learning.
* Limits deep neural network training.

---

### **3. Convolutional Neural Networks (CNNs)**

* Designed for **image data**.
* Use filters to detect patterns like edges, shapes.
* Widely used in computer vision.

---

### **4. Transformers**

* Modern architecture for sequence data (text).
* Uses **attention mechanism** instead of recurrence.
* Handles long-range dependencies better.

---

### **5. Self-Attention Mechanism**

* Helps model understand which words are important.
* Each word looks at other words in the sentence.
* Captures context effectively.

---

### **6. Multi-Head Attention**

* Multiple attention layers run in parallel.
* Each learns different relationships.
* Improves model understanding.

---

### **7. Positional Encoding**

* Adds order information to input tokens.
* Needed because transformers process data in parallel.
* Without it → model doesn’t know sequence order.

---

### **8. Word Embeddings**

* Converts words into vectors (numbers).
* Similar words have similar vectors.
* Captures meaning.

---

### **9. Encoder-Decoder Architecture**

* Encoder processes input.
* Decoder generates output.
* Used in translation, summarization.

---

### **10. Fine-tuning**

* Adapting a pretrained model to a specific task.
* Uses smaller dataset.
* Saves time and improves performance.

---

### **11. Recurrent Neural Networks (RNNs)**

* Have memory (past inputs influence current output).
* Good for sequence data (text, time series).
* Uses hidden state.

---

* First and most important step

* Helps prioritize risks

* Taking actions to reduce risk
* Includes prevention, mitigation, transfer
* Final step in process

* Used in decision-making

* Difference between estimated risk and actual risk
* Comes from uncertainty or wrong assumptions
* Important in risk models

* **Organizational** (company-level)
* **National** (society-level)

# **1. Concepts (simple + full coverage)**

### **1. Interpretability vs Explainability**

* **Interpretability** → model itself is understandable (like linear regression).
* **Explainability** → using extra methods to explain complex models.
* Simple = interpretable, Complex = needs explanation.

---

### **2. Black Box Models**

* Models like deep neural networks.
* Internal working not understandable by humans.
* High accuracy but low transparency.

---

### **3. Need for Explainable AI (XAI)**

* Required for:

  * Trust
  * Accountability
  * Legal compliance
* Important in healthcare, finance.

---

### **4. Transparency in AI**

* Ability to understand how decisions are made.
* Helps debugging and trust.

---

### **5. Feature Attribution**

* Shows which input features influenced output.
* Example: which pixel or word mattered most.

---

### **6. Saliency Maps**

* Highlight important parts of input.
* Used in images (important pixels).
* Limitation: shows “where” not “how”.

---

### **7. Perturbation Methods**

* Change input slightly and observe output.
* Helps understand model behavior.

---

### **8. LIME (Local Interpretable Model-Agnostic Explanations)**

* Explains single prediction locally.
* Uses simpler model around data point.

---

### **9. SHAP (SHapley Additive Explanations)**

* Based on game theory.
* Shows contribution of each feature.
* More consistent than LIME.

---

### **10. Local vs Global Explanation**

* Local → explains one prediction
* Global → explains entire model behavior

---

### **11. Fidelity in Explanation**

* How accurate explanation reflects real model
* Low fidelity → misleading explanation

---

### **12. Black Box Problem**

* Hard to know why model made decision
* Risk in critical applications

---

### **13. Epistemic Problem**

* Humans don’t understand model logic
* Due to complexity or lack of knowledge

---

### **14. “Many Things” Problem**

* System failures due to multiple interacting factors
* Hard to identify single cause

---

### **15. Tradeoff: Accuracy vs Interpretability**

* Complex models → accurate but opaque
* Simple models → interpretable but less powerful

---

### **16. Rudin’s Argument**

* No need to sacrifice interpretability for accuracy
* Interpretable models can perform equally well

---

### **17. Model Debugging using XAI**

* Helps find errors in model
* Identifies wrong patterns learned

---

### **18. Ethical Importance of XAI**

* Ensures fairness
* Prevents discrimination
* Enables accountability

---

# **1. Concepts (simple + full coverage)**

### **1. Sources of Bias in AI**

* Bias can enter at multiple stages of AI pipeline.
* Not just model → also data, labeling, and deployment.
* Important to identify where bias originates.

---

### **2. Historical Bias**

* Exists in real-world data before AI.
* AI learns existing social inequalities.
* Example: hiring bias in past data.

---

### **3. Representation Bias**

* Occurs when dataset is not representative.
* Some groups under/over-represented.
* Leads to unfair predictions.

---

### **4. Measurement Bias**

* Happens during data collection/labeling.
* Labels may be inaccurate or subjective.
* Affects model learning.

---

### **5. Aggregation Bias**

* One model applied to diverse groups.
* Assumes same behavior for all.
* Reduces fairness.

---

### **6. Deployment Bias**

* Occurs when model used in wrong context.
* Humans may misinterpret outputs.
* Happens after model is built.

---

### **7. “More Data” Myth**

* More data doesn’t always fix bias.
* If labels are biased → problem remains.
* Data quality matters more than quantity.

---

### **8. Annotator Bias**

* Human labelers introduce bias.
* Their opinions affect dataset.
* Impacts model behavior.

---

### **9. Robustness in AI**

* Ability to perform well in different conditions.
* Includes:

  * Different users
  * Different environments
* Important for real-world systems.

---

### **10. Overfitting**

* Model learns noise instead of pattern.
* Performs well on training but плохо on real data.
* Causes poor generalization.

---

### **11. Spurious Correlation**

* False relationships in data.
* Model learns wrong patterns.
* Leads to incorrect predictions.

---

### **12. AI Hallucination**

* Model generates false but confident output.
* Happens in LLMs.
* Caused by poor training or complexity.

---

### **13. Jailbreaking**

* User tricks AI to bypass restrictions.
* Exploits model weaknesses.
* Security issue.

---

### **14. Prompt Injection**

* Malicious instructions hidden in input.
* Model follows wrong instructions.
* Dangerous in AI systems.

---

### **15. Indirect Prompt Injection**

* Attack hidden in external data (emails, docs).
* Model unknowingly executes it.
* Hard to detect.

---

### **16. Model Stealing**

* Attacker extracts model behavior.
* Uses repeated queries.
* Steals intellectual property.

---

### **17. Adversarial Attacks (Revisited)**

* Small input change → wrong output.
* Exploits model sensitivity.

---

### **18. Data Poisoning (Revisited)**

* Training data manipulated.
* Model learns incorrect patterns.

---

### **19. Bias vs Variance Tradeoff**

* Bias → oversimplified model
* Variance → overfitting
* Need balance for best performance

---

### **20. Explainability Importance**

* Required for:

  * Trust
  * Regulation
  * Debugging
* Especially in critical systems

---

### **1. Fairness in AI**

* Fairness means AI decisions should not discriminate between groups.
* Important in hiring, loans, healthcare.
* Ensures equal treatment.

* **Individual fairness** → similar people treated similarly
* **Group fairness** → different groups treated equally
* Both are important but hard to balance

---

### **3. Equality of Opportunity**

* People should have equal chance regardless of background.
* Focus on fairness in outcomes.

---

### **4. Narrow View of Equality**

* Only current qualifications matter.
* Ignores past disadvantages.

---

### **5. Middle View**

* Supports interventions like affirmative action.
* Helps reduce inequality at decision stage.

---

### **6. Broad View**

* Focuses on root causes in society.
* Long-term fairness approach.

---

### **7. Systematic Disadvantage**

* Some groups face disadvantages due to social factors.
* Not related to ability or performance.

---

### **8. Discrimination in AI**

* Happens when model treats groups unfairly.
* Can be intentional or unintentional.

---

### **9. Unintentional Bias**

* AI may produce unfair results without intention.
* Comes from biased data or design.

---

### **10. Diversity vs Inclusion**

* **Diversity** → different people included
* **Inclusion** → equal participation and access

---

### **11. Equity vs Equality**

* **Equality** → same treatment
* **Equity** → fair treatment based on need

---

### **12. Justice as Fairness (Rawls)**

* Fair distribution of resources
* Based on impartial decisions

---

### **13. Ethical Responsibility in AI**

* AI cannot be morally responsible
* Humans must take responsibility

---

### **14. Responsibility Gap**

* Happens when no one is clearly responsible for AI decisions
* Due to automation and complexity

---

### **15. Distributed Responsibility**

* Many people involved in AI system
* Responsibility shared across them

---

### **16. Many Hands Problem**

* Hard to assign blame when many actors involved
* Common in AI failures

---

### **17. Many Things Problem**

* Failure caused by multiple factors
* Difficult to identify root cause

---

### **18. Transparency in Ethics**

* Important for accountability
* Users should understand decisions

---

### **19. Importance of Fair AI**

* Prevents discrimination
* Builds trust
* Required for regulation

* Safety

def wrap_text(text, max_width, font_size, weight="regular"):
    text = re.sub(r"\s+", " ", text).strip()

class BlockParser(HTMLParser):
    BLOCK_TAGS = {"h1", "h2", "h3", "h4", "p", "div"}
    SKIP_TAGS = {"style", "script", "head", "title"}

    def __init__(self):
        super().__init__()
        self.blocks = []
        self.skip_depth = 0
        self.in_body = False
        self.current = None

    def handle_starttag(self, tag, attrs):
        attrs = dict(attrs)
        if tag == "body":
            self.in_body = True
            return
        if tag in self.SKIP_TAGS:
            self.skip_depth += 1
            return
        if not self.in_body or self.skip_depth:
            return
        if tag in self.BLOCK_TAGS:
            self.current = {"tag": tag, "class": attrs.get("class", ""), "parts": []}
        elif tag == "br" and self.current:
            self.current["parts"].append("\n")

    def handle_endtag(self, tag):
        if tag == "body":
            self.in_body = False
            return
        if tag in self.SKIP_TAGS and self.skip_depth:
            self.skip_depth -= 1
            return
        if not self.in_body or self.skip_depth:
            return
        if self.current and tag == self.current["tag"]:
            text = "".join(self.current["parts"])
            text = html.unescape(text).replace("\xa0", " ")
            lines = [re.sub(r"\s+", " ", line).strip() for line in text.split("\n")]

            if text or self.current["tag"] == "div":
                self.blocks.append({"tag": self.current["tag"], "class": self.current["class"], "text": text})

    def handle_data(self, data):
        if self.current and self.in_body and not self.skip_depth:
            self.current["parts"].append(data)


class PDFWriter:
    def __init__(self):
        self.objects = []

    def add_object(self, data):
        self.objects.append(data)
        return len(self.objects)

    def write(self, pages, output_path):
        font_regular = self.add_object("<< /Type /Font /Subtype /Type1 /BaseFont /Helvetica >>")
        font_bold = self.add_object("<< /Type /Font /Subtype /Type1 /BaseFont /Helvetica-Bold >>")
        font_italic = self.add_object("<< /Type /Font /Subtype /Type1 /BaseFont /Helvetica-Oblique >>")
        pages_ref = self.add_object("")

        page_refs = []
        for page in pages:
            stream = "\n".join(page).encode("latin-1", errors="replace")
            content_ref = self.add_object(f"<< /Length {len(stream)} >>\nstream\n{stream.decode('latin-1')}\nendstream")
            page_ref = self.add_object(
                f"<< /Type /Page /Parent {pages_ref} 0 R /MediaBox [0 0 {PAGE_W} {PAGE_H}] "
                f"/Resources << /Font << /F1 {font_regular} 0 R /F2 {font_bold} 0 R /F3 {font_italic} 0 R >> >> "
                f"/Contents {content_ref} 0 R >>"
            )
            page_refs.append(page_ref)

        kids = " ".join(f"{ref} 0 R" for ref in page_refs)
        self.objects[pages_ref - 1] = f"<< /Type /Pages /Count {len(page_refs)} /Kids [ {kids} ] >>"
        catalog_ref = self.add_object(f"<< /Type /Catalog /Pages {pages_ref} 0 R >>")

        result = [b"%PDF-1.4\n%\xe2\xe3\xcf\xd3\n"]
        offsets = [0]
        current = len(result[0])
        for idx, obj in enumerate(self.objects, start=1):
            encoded = f"{idx} 0 obj\n{obj}\nendobj\n".encode("latin-1", errors="replace")
            offsets.append(current)
            result.append(encoded)
            current += len(encoded)

        xref_offset = current
        result.append(f"xref\n0 {len(self.objects) + 1}\n".encode("latin-1"))
        result.append(b"0000000000 65535 f \n")
        for off in offsets[1:]:
            result.append(f"{off:010d} 00000 n \n".encode("latin-1"))
        result.append(
            (
                f"trailer\n<< /Size {len(self.objects) + 1} /Root {catalog_ref} 0 R >>\n"
                f"startxref\n{xref_offset}\n%%EOF\n"
            ).encode("latin-1")
        )
        output_path.write_bytes(b"".join(result))

        if self.y - height < BOTTOM_MARGIN:

        cmd = [color_cmd(fill), f"{x:.2f} {y:.2f} {w:.2f} {h:.2f} re"]
        if stroke:
            cmd.insert(1, color_cmd(stroke, stroke=True))
            cmd.insert(2, f"{stroke_width:.2f} w")
            cmd.append("B")
        else:
            cmd.append("f")
        self.page.append("\n".join(cmd))

        self.page.append(
            "\n".join(
                [
                    color_cmd(stroke, stroke=True),
                    f"{stroke_width:.2f} w",
                    f"{x1:.2f} {y1:.2f} m",
                    f"{x2:.2f} {y2:.2f} l",
                    "S",
                ]
            )

        self.rect(MARGIN_X, PAGE_H - 36, 96, 4, BLUE)

    def draw_footer(self):
        self.line(MARGIN_X, 28, PAGE_W - MARGIN_X, 28, LINE, 1)
        self.text(MARGIN_X, 18, self.title, 9, SLATE)
        self.text(PAGE_W - MARGIN_X - 40, 18, f"Page {self.page_number}", 9, SLATE, align="right")

    def ensure_page(self):
        if not self.started:
            self.new_page()

    def need_space(self, height):
        self.ensure_page()
        if self.y - height < BOTTOM_MARGIN:
            self.new_page()

    def rect(self, x, y, w, h, fill, stroke=None, stroke_width=1):
        cmd = [color_cmd(fill), f"{x:.2f} {y:.2f} {w:.2f} {h:.2f} re"]
        if stroke:
            cmd.insert(1, color_cmd(stroke, stroke=True))
            cmd.insert(2, f"{stroke_width:.2f} w")
            cmd.append("B")
        else:
            cmd.append("f")
        self.page.append("\n".join(cmd))

    def line(self, x1, y1, x2, y2, stroke, stroke_width=1):
        self.page.append(
            "\n".join(
                [
                    color_cmd(stroke, stroke=True),
                    f"{stroke_width:.2f} w",
                    f"{x1:.2f} {y1:.2f} m",
                    f"{x2:.2f} {y2:.2f} l",
                    "S",
                ]
            )
        )

    def text(self, x, y, text, size, color, font="F1", align="left"):
        width = estimate_width(text, size, "bold" if font == "F2" else "italic" if font == "F3" else "regular")
        if align == "center":
            x = x - (width / 2)
        elif align == "right":
            x = x - width
        self.page.append(
            "\n".join(
                [
                    "BT",
                    color_cmd(color),
                    f"/{font} {size} Tf",
                    f"1 0 0 1 {x:.2f} {y:.2f} Tm",
                    f"({pdf_escape(text)}) Tj",
                    "ET",
                ]
            )
        )

    def paragraph(self, lines, size=10.5, leading=14, color=TEXT, font="F1", x=MARGIN_X, extra_top=0, extra_bottom=0):
        total_h = extra_top + (len(lines) * leading) + extra_bottom
        self.need_space(total_h)
        self.y -= extra_top
        for line in lines:
            self.text(x, self.y, line, size, color, font=font)
            self.y -= leading
        self.y -= extra_bottom

    def render_title_page(self, title, subtitle):
        self.new_page()
        self.rect(0, PAGE_H - 170, PAGE_W, 140, NAVY)
        self.rect(MARGIN_X, PAGE_H - 190, 140, 6, WHITE)
        self.text(PAGE_W / 2, PAGE_H - 105, title, 24, WHITE, font="F2", align="center")
        subtitle_lines = wrap_text(subtitle, CONTENT_W - 40, 12)
        y = PAGE_H - 132
        for line in subtitle_lines:
            self.text(PAGE_W / 2, y, line, 12, (220, 231, 242), align="center")
            y -= 18

        info = [
            "Weeks 1 to 12",
            "Concepts explained in simple language",
            "Exam-focused MCQs with answers",
        ]
        y = PAGE_H - 255
        for line in info:
            self.text(PAGE_W / 2, y, line, 13, NAVY, font="F2" if line == "Weeks 1 to 12" else "F1", align="center")
            y -= 24

        self.rect(MARGIN_X, PAGE_H - 390, CONTENT_W, 110, LIGHT_BLUE)
        note_lines = wrap_text(
            "This PDF is arranged week by week. Each week contains a Concepts section with short, clear explanations and a full MCQs section written in exam-friendly language.",
            CONTENT_W - 36,
            11,
        )
        y = PAGE_H - 320
        for line in note_lines:
            self.text(MARGIN_X + 18, y, line, 11, TEXT)
            y -= 16
        self.y = 180

    def render_week_header(self, text):
        self.new_page()
        self.rect(MARGIN_X, self.y - 28, CONTENT_W, 34, NAVY)
        self.text(MARGIN_X + 18, self.y - 8, text, 18, WHITE, font="F2")
        self.y -= 48

    def render_section_header(self, text):
        lines = wrap_text(text, CONTENT_W - 24, 13, "bold")
        height = 16 + len(lines) * 16
        self.need_space(height)
        self.rect(MARGIN_X, self.y - height + 4, CONTENT_W, height, LIGHT_BLUE)
        line_y = self.y - 14
        for line in lines:
            self.text(MARGIN_X + 12, line_y, line, 13, NAVY, font="F2")
            line_y -= 16
        self.y -= height + 6

    def render_subsection_header(self, text):
        lines = wrap_text(text, CONTENT_W, 12, "bold")
        self.paragraph(lines, size=12, leading=15, color=BLUE, font="F2", extra_top=6, extra_bottom=2)

    def render_normal_paragraph(self, text, class_name=""):
        if not text:
            self.y -= 4
            return
        if class_name == "question":
            lines = wrap_text(text, CONTENT_W - 12, 11, "bold")
            self.paragraph(lines, size=11, leading=14, color=NAVY, font="F2", extra_top=8, extra_bottom=1)
        elif class_name == "options":
            option_lines = []
            for part in text.split("\n"):
                option_lines.extend(wrap_text(part, CONTENT_W - 28, 10.5))
            self.paragraph(option_lines, size=10.5, leading=13, color=TEXT, x=MARGIN_X + 18, extra_top=0, extra_bottom=2)
        elif class_name == "answer":
            lines = wrap_text(text, CONTENT_W - 18, 10.5, "bold")
            self.paragraph(lines, size=10.5, leading=13, color=GREEN, font="F2", x=MARGIN_X + 8, extra_top=0, extra_bottom=5)
            self.line(MARGIN_X, self.y + 3, PAGE_W - MARGIN_X, self.y + 3, LINE, 0.7)
            self.y -= 6
        elif class_name == "example":
            lines = wrap_text(text, CONTENT_W - 24, 10.5, "italic")
            height = 14 + len(lines) * 14
            self.need_space(height + 6)
            self.rect(MARGIN_X, self.y - height + 3, CONTENT_W, height, SOFT_GOLD)
            yy = self.y - 11
            for line in lines:
                self.text(MARGIN_X + 12, yy, line, 10.5, TEXT, font="F3")
                yy -= 14
            self.y -= height + 8
        elif class_name == "intro":
            lines = wrap_text(text, CONTENT_W - 40, 12)
            total_h = len(lines) * 16 + 10
            self.need_space(total_h)
            for line in lines:
                self.text(PAGE_W / 2, self.y, line, 12, SLATE, align="center")
                self.y -= 16
            self.y -= 8
        else:
            blocks = text.split("\n")
            for block in blocks:
                lines = wrap_text(block, CONTENT_W, 10.5)
                self.paragraph(lines, size=10.5, leading=14, color=TEXT, extra_top=0, extra_bottom=2)

    def render(self):
        title = None
        subtitle = None
        remaining = []
        for block in self.blocks:
            if block["tag"] == "h1" and title is None:
                title = block["text"]
                self.title = title
                continue
            if block["class"] == "intro" and subtitle is None:
                subtitle = block["text"]
                continue
            remaining.append(block)

        self.render_title_page(title or self.title, subtitle or "Weeks 1 to 12")

        for block in remaining:
            tag = block["tag"]
            cls = block["class"]
            text = block["text"]

            if tag == "div" and cls == "page-break":
                continue
            if tag == "h2":
                self.render_week_header(text)
            elif tag == "h3":
                self.render_section_header(text)
            elif tag == "h4":
                self.render_subsection_header(text)
            elif tag == "p":
                self.render_normal_paragraph(text, cls)

        self.finish_page()
        return self.pages
